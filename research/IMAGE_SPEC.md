# RoboIndex 卡片图片标准

这份文档是收集 entity 图片的目标规格。所有新增和重做的图都按这个标准走。

## 通用硬规则

| 项 | 规则 |
|---|---|
| 宽高比 | **1:1 正方形**（适配 wheel selector 卡片） |
| 尺寸 | **1000×1000 px** 优先；800×800 起步可接受 |
| 命名 | `images/<partition>/<entity-id-slug>.{jpg\|png}` —— 例 `images/hardware/figure-03.jpg` |
| 路径 | repo 相对路径，渲染时走 `resolveImageUrl()` 加上 `/RoboIndex/` base |
| JSON 字段 | `imageUrl: "images/hardware/figure-03.jpg"`（不带前导斜杠） |
| 文件大小 | JPG ≤ 200 KB，PNG ≤ 400 KB（如需更大压缩即可，retina 也够） |

## 4 种图片原型

按内容性质分，规则少、好判断。

### 原型 A：产品 hero 图（实体硬件）

**用于**：整机平台、灵巧手 & 夹爪、机械臂、关节模组、核心零部件、传感器、计算平台、能源动力

**视觉规则**：
- 单一物体居中
- **白底或浅灰底**（不要黑底，不要彩色底；浅渐变可接受）
- 正面或 3/4 视角
- 物体四周留约 **10% padding**
- 不要水印、不要价格标、不要型号文字叠层
- 优先用厂商发布会/datasheet/官网产品页的官方图

**Humanoid 特别注意**：站立姿势，不要动作中、不要 demo 现场、不要与人合影。Tesla Optimus/Figure 03/Atlas Electric 厂商发布的标准 hero 就是这个风格。

**文件格式**：JPG

---

### 原型 B：合成标题卡（软件 / 抽象物）

**用于**：基础模型、算法框架、控制算法、仿真平台、数据集、评测基准、开发生态

**视觉规则**：
程序生成的统一风格标题卡，包含：
- 大字号 entity name（中英对照如有）
- 副标题：作者/lab/公司名
- 角落小字：年份 + category
- 渐变背景（按 category 分配固定色系，详见下表）
- 居中布局，无图标杂物

**色系映射**（建议）：

| Category | 色系 |
|---|---|
| 基础模型 | 深蓝 → 紫 |
| 算法框架 | 深绿 → 青 |
| 控制算法 | 橙 → 红 |
| 仿真平台 | 深紫 → 粉 |
| 数据集 | 灰 → 蓝灰 |
| 评测基准 | 黄 → 棕 |
| 开发生态 | 青 → 绿 |

**文件格式**：PNG（保边缘清晰）

**实现路径**：写一个 Python 脚本根据 entity JSON 批量生成，避免人工逐个画。详见下方"待办：合成卡生成器"。

---

### 原型 C：Logo 标志（参与实体）

**用于**：产业、资本、实验室

**视觉规则**：
- **纯 logo，透明 PNG**（卡片底色透出，与 entity 卡片设计统一）
- 居中，四周留 ~15% padding
- 不要立体阴影、不要倾斜、不要文字版加图标版混用
- 一个组织只放一个 logo（不要图标+文字两个版本拼）
- 中英文双 logo 时优先英文版

**素材来源**：
- 公司：官网 footer 或 press kit
- VC：Crunchbase/官网
- 大学/lab：官方品牌识别页面
- 找不到官方版的可用 Wikipedia 高清版本

**文件格式**：PNG（必须透明）

---

### 原型 D：部署现场（应用场景）

**用于**：应用场景；数采 & 遥操（当展示人佩戴设备时）

**视觉规则**：
- 真实场景：机器人/设备 + 环境
- 横向构图可接受但裁成 1:1
- 优先官方部署案例图，次选 IEEE Spectrum/Robot Report 等权威媒体配图
- 避免：合成图、概念渲染图、AI 生成图

**文件格式**：JPG

## 缺图 fallback（短期内无法收集到的条目）

不留空——空卡片让用户以为是 bug。规则：

1. **自动用原型 B 的合成标题卡生成占位**（同样从 entity JSON 拿信息）
2. 右上角加一个小徽章 **「待补图」** 让缺口可见
3. 鼠标 hover 时 tooltip 说明这是 placeholder，鼓励社区贡献

这个 fallback 在 EntityCard.tsx 渲染时生效——不需要往 JSON 里塞假 imageUrl。

## 反例（**不要**做的）

- ❌ Unsplash/Pexels 的人形机器人 stock 照（早期 placeholder 错误，audit.md 已记录）
- ❌ AI 文生图（不可信、不可溯源、违反 source attribution 原则）
- ❌ 把多个产品代际拼在一张图（演进链已经有专门 UI 显示）
- ❌ 黑底产品图（与卡片浅色背景冲突）
- ❌ 加水印或厂商发布会标语叠层的图
- ❌ 倾斜 logo（要正放）
- ❌ logo 立体化处理（要平面）

## 待办（后续可独立实现）

1. **合成标题卡生成器** —— Python 脚本，输入 `public/data/*.json`，输出 `public/images/<partition>/<id>.png`。无图条目自动跑这个补。需要的依赖：Pillow。颜色按 category 色系映射。预估 100 行代码。
2. **EntityCard.tsx 加「待补图」徽章** —— 当 imageUrl 缺失时叠加。约 10 行 JSX。
3. **图片预处理脚本** —— 收集到的原图自动裁 1:1、压到 1000×1000、转 JPG/PNG。基于 Pillow。
4. **图片审计 CLI** —— 检查每个 entity 的 imageUrl 是否存在、是否符合原型规则（尺寸、底色等）。
